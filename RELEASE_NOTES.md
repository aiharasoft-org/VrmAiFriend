# v0.8.0 — Windows 対応

**リリース名:** Windows 対応  
**対象プラットフォーム:** **Windows**、**macOS**（Windows 版を新規追加）

**Windows 版** が初めてリリースされました。macOS 版は従来どおり利用できます。あわせて、音声の話速調整、マイク入力の安定化、設定タブでのマイク状態表示、OS 別のカメラ操作ガイドなど、Windows 移植に伴う UX を整えています。

## ハイライト

- **Windows 対応（本リリースの主題）** — 初の Windows ビルド。Aivis Speech の検出・起動、透過ウィンドウ
- **話速スライダー** — 音声タブで 0.8〜1.5 を調整（既定 1.0、永続化）
- **設定タブのマイク状態** — 物理マイクの有無・権限・入力状態を表示
- **マイク入力の安定化** — 高サンプルレート取得、自動再試行、感度の自動更新
- **マイクメータ改善** — 実測に合わせた動的スケールと閾値表示
- **OS 別カメラガイド** — Windows: Alt / Ctrl、macOS: Option / Cmd
- **Windows CPU 向け Tips** — README に電源モード「最適なパフォーマンス」の案内

## インストール

### 共通（Windows / macOS）

| 項目 | 内容 |
| ---- | ---- |
| ダウンロード | [VrmAiFriend](https://github.com/aiharasoft-org/VrmAiFriend) |
| Aivis Speech Engine | 必須。[AivisSpeech Engine](https://github.com/Aivis-Project/AivisSpeech-Engine) をインストールし起動 |
| Gemini API キー | **設定** タブで保存 |
| 物理マイク | 必須（仮想オーディオのみでは不可） |

### Windows

1. リリースページから **Windows 版** をダウンロードし、ZIP を展開する。
2. フォルダ内の **`.exe`** を実行する。

**SmartScreen / セキュリティ警告（未署名アプリ）**

GitHub から取得した Windows 版はコード署名がないため、初回起動時に **Microsoft Defender SmartScreen** がブロックすることがあります（「Windows によって PC が保護されました」「不明な発行元」など）。

信頼できる配布元であることを確認したうえで、次のいずれかで起動してください。

| 方法 | 手順 |
| ---- | ---- |
| 起動ダイアログ | **詳細情報** → **実行** |
| ブロック解除（推奨） | `.exe` の **プロパティ** → **全般** → **ブロックの解除** にチェック → **OK** → 再実行 |

ブラウザやウイルス対策ソフトがダウンロード・実行を止める場合も、配布元を確認のうえ **保持** / **許可** してください。詳細は [README.md](README.md) の **Windows** 節を参照。

**Aivis Speech（Windows）:** [AivisSpeech](https://github.com/Aivis-Project/AivisSpeech) 本体または Engine のいずれかをインストール。**Engine 接続 OK** なら利用可能。音声タブの **Aivisを開く** から起動できる。

### macOS

1. リリースページから **macOS 版** をダウンロードする。
2. `.app` を起動する。Gatekeeper で止まる場合は **Control + クリック** → **開く**、または **システム設定** → **プライバシーとセキュリティ** から許可する。
3. **AivisSpeech Engine** をインストールし、**マイク** 権限を OS 設定で許可する。

## Windows 対応（機能）

| 項目 | 内容 |
| ---- | ---- |
| 対象 OS | Windows、macOS |
| Aivis Speech | Windows では `AivisSpeech.exe` / Engine のパスを検出 |
| 起動 | 音声タブの **Aivisを開く** から Windows 版を起動 |
| 透過ウィンドウ | Windows ビルドでも `ESC` で透過切替 |

## 音声（話速・Aivis）

- **話速** スライダー: **0.8** 〜 **1.5**（中央が現在値、既定 **1.0**）
- 話者・スタイルとあわせて次回起動時に復元
- **音声を初期値に戻す** で話速も 1.0 に戻る
- Aivis 合成時に `speedScale` を反映（先読み合成で 2 文目以降の待ちを短縮）

### Windows で音声合成が遅い場合（CPU モード）

Aivis Speech Engine を CPU モードで使い「少し遅い」と感じる場合:

1. **設定** → **システム** → **電源**
2. **電源モード** を **最適なパフォーマンス** に変更

E コアから P コアへ処理が割り当てられやすくなり、生成が速くなることがあります。

## マイク感度・レベルメータ

- マイク入力の **最大値・閾値・エコー** からメータ尺度を動的に決定
- 通常の声でメータが自然に動き、調整済みの **赤線（閾値）** がノイズと声の間付近に表示されやすくなった
- **マイク感度調整** — スピーカー利用時は手動調整を推奨。未調整時は周囲ノイズを観測して自動更新を試みる
- 48kHz / 44.1kHz で取得し 16kHz へリサンプル。開始失敗時は定期再試行し、物理マイクの接続・復帰を検知

## 設定タブのマイク状態

**設定** タブでは GEMINI → VRoid Hub → **マイク** → Aivis Speech の順で状態を表示します。

| 表示 | 意味 |
| ---- | ---- |
| `OK` | 物理マイクから音声を取得中 |
| `準備完了` | 物理マイクあり・権限 OK（未キャプチャ） |
| `物理マイクが見つかりません` | 仮想オーディオ（VB-Audio 等）のみ、または実マイクなし |
| `権限なし` | OS のマイク権限が未許可 |
| `他アプリが使用中の可能性` | デバイスはあるが音声が来ない |
| `初期化中...` / `権限確認中...` | 起動直後など |

> **Windows:** VB-Audio Virtual Cable などの仮想デバイスだけが入っている PC では、物理マイクが無いと判断され **物理マイクが見つかりません** と表示されます。

## カメラ操作（OS 別）

| 操作 | macOS | Windows |
| ---- | ----- | ------- |
| ズーム（キー） | `Cmd` + ↑/↓ | `Ctrl` + ↑/↓ |
| 高さ・チルト・左右・ズーム（微調整） | `Option` + 各キー | `Alt` + 各キー |

**カメラ** タブのガイド画像も実行 OS に応じて自動切替されます。

## メニュー（v0.8 時点）

v0.7 からタブ構成の大きな変更はありません。起動時の既定タブは **設定** です。

| タブ | 追記・変更点 |
| ---- | ------------ |
| **設定** | マイク状態を追加（GEMINI / VRoid Hub / マイク / Aivis の順） |
| **音声** | 話速スライダー追加 |
| **カメラ** | OS 別ガイド画像 |
| **About / ヘルプ** | OS 別キー表記 |

## 必要な環境

| 項目 | 内容 |
| ---- | ---- |
| OS | **Windows** または **macOS** |
| 物理マイク | 必須（仮想オーディオのみでは利用不可） |
| Aivis Speech Engine | 音声合成に必須 |
| Gemini API キー | **設定** タブで保存 |

## アップグレード時の注意（v0.7 → v0.8）

- **Windows 版は新規対応**（本リリースの主題）。macOS 版の既存データ（記憶・好感度・音声設定など）は同じ形式で利用可能
- Windows 版は [VrmAiFriend](https://github.com/aiharasoft-org/VrmAiFriend) から入手。SmartScreen 等の警告は README の **Windows** 節を参照
- 話速は未設定時 **1.0**。旧 `aivis_voice_selection.json` に話速が無い場合も 1.0 として扱う
- マイク感度を調整済みの場合、**マイク感度調整** のやり直しを推奨（メータ・閾値表示の改善のため）
- 仮想オーディオ（VB-Audio 等）のみの環境では **物理マイクが見つかりません** と表示される
- カメラ操作の Windows 修飾キー（Ctrl / Alt）に合わせてガイドが切り替わる

# v0.7.0 — 会話の表情と好感度システム

**リリース名:** 会話の表情と好感度システム

会話のトーンに合わせてキャラクターの **表情が滑らかに変わる** ようになりました。あわせて、会話の積み重ねで変化する **好感度（0〜100）** を導入し、AI の接し方にも反映されます。メニューは **設定 / キャラ / 感情** などに再構成され、API キーの保存や記憶の確認もアプリ内で行えます。

## ハイライト

- **会話連動の表情** — 返答の感情に合わせて VRM の表情が変化し、切り替えは滑らか
- **好感度システム** — 会話の感情に応じて 0〜100 が変動し、口調・態度に反映
- **感情タブ** — 現在の表情と好感度をメニューから確認
- **キャラタブの記憶** — 「記憶を見る」で覚えている内容を確認、「記憶を削除」で消去
- **設定タブ** — Gemini API キーをアプリ内に暗号化保存
- **人格パラメータ 5 段階化** — やさしさ・言葉遣いなど 7 項目を 1〜5 の段階で設定
- **ダンス 4 種と即時停止** — 超絶かわいいほか 3 種を追加。「やめて」でダンス・ポーズを停止

## 会話連動の表情

AI が返答するたびに、話のトーンに合った表情が適用されます。表情の切り替えはフェードで滑らかに行われます。

| 表情 | 例 |
| ---- | -- |
| happy | 嬉しい・楽しい話題 |
| sad | 悲しい・残念な話題 |
| angry | 怒り・不満 |
| surprised | 驚き |
| relaxed | 穏やか・リラックス |
| neutral | 特に強い感情がないとき |

タッチリアクション中はタッチ側の表情が優先されます。モデルによっては未対応の表情がある場合があります。

## 好感度（親密度レーティング）

好感度は **0〜100** で管理され、会話の感情が確定したタイミングで自動的に変動します。

| 範囲 | 目安 |
| ---- | ---- |
| 0〜19 | よそよそしい |
| 20〜39 | やや距離感がある |
| 40〜59 | 普通の友人 |
| 60〜79 | 親しみを込めた |
| 80〜100 | 好意を率直に表現 |

- **happy** の感情が確定すると +1、**sad** だと −1（同一ターン内の重複は抑制）
- 段階が変わると、AI の口調・態度が更新される
- **感情** タブでメーターを確認でき、**好感度を初期値に戻す** でリセットできる

## メニューの変更（v0.7）

`ESC` でメニュー表示。左上の **▲** で折りたたみ。

| タブ | 内容 |
| ---- | ---- |
| **設定** | API キー保存、Gemini Live・Hub・Aivis の接続状態 |
| **モデル** | VRoid Hub モデル、同梱モデルへの復帰 |
| **キャラ** | 名前・性格の編集・保存、記憶の確認・削除 |
| **音声** | Aivis の話者・マイク感度調整 |
| **話題** | AI 主導トークの設定 |
| **感情** | 現在の表情・好感度 |
| **カメラ** | カメラ操作・リセット |
| **About** | バージョン・ヘルプ |

About の **初期設定に戻す**（一括リセット）は廃止しました。各タブの **初期値に戻す** で項目ごとにリセットしてください。

## キャラ設定と記憶

1. **キャラ** タブで基本情報と性格パラメータ（**1〜5 段階**）を編集する。
2. **保存** を押すと設定が保存され、会話に反映される（接続中は再接続）。
3. **キャラを初期値に戻す** — 名前・性格のみリセット（**記憶は残る**）。
4. **記憶を見る** — 覚えているプロフィールや出来事を確認。
5. **記憶を削除** — 確認後に記憶を消去。

v0.6 以前の 0〜10 スケールの性格設定は、読み込み時に 1〜5 段階へ自動変換されます。

## ダンス

| 話しかけ例 | ダンス |
| ---------- | ------ |
| コンコンダンス | コンコンダンス |
| 超絶かわいい、かわいい | 超絶かわいい |
| キャラメルダンス | キャラメルダンス |
| ウマウマパラパラ | ウマウマパラパラ |
| ストップ、やめて | ダンス・ポーズの停止 |

## 操作の変更（v0.7）

v0.6 からのキーバインド変更はありません。

| キー | 動作 |
| ---- | ---- |
| `Option` + スペース | マイク送信のミュート切替 |
| `F` | 視線追従の固定 / 解除 |
| `P` | ポーズ撮影モードの開始 / 終了 |
| `ESC` | メニュー表示とウィンドウ透過の切替 |

## 必要な環境

| 項目 | 内容 |
| ---- | ---- |
| Aivis Speech Engine | 音声合成に必須 |
| Gemini API キー | **設定** タブで保存（`~/.env` からの取り込みにも対応） |

## アップグレード時の注意

- v0.6.0 からの追加要件はありません（Aivis + Gemini API キーは従来どおり）
- API キーは **設定** タブから保存できます（`~/.env` のキーは初回に取り込まれる場合があります）
- 性格パラメータは 1〜5 段階に変わりました（旧値は自動変換）
- 好感度は新規に 0 から始まります（**感情** タブからリセット可能）
- About の一括リセットはなくなりました。各タブのボタンで個別にリセットしてください
- 記憶の確認は **キャラ** タブの **記憶を見る** から行えます

# v0.6.0 — 長期記憶と人格設定のアプリ内統合

**リリース名:** 長期記憶と人格設定のアプリ内統合

会話の内容からユーザーのことを覚え、次の会話に活かす **長期記憶** を追加しました。あわせて、キャラクター設定の編集を **メニューの「人格」タブ** に統合しました。

## ハイライト

- **長期記憶** — 会話からプロフィール・事実を覚え、以降の会話に反映
- **人格タブ** — 基本情報・性格をアプリ内で編集・保存（未接続時も表示）
- **マイク感度調整とエコーゲート** — スピーカー利用時のエコー対策、AI 発話中の割り込み
- **音声全二重化** — AI が話している最中もマイク送信・割り込みが可能
- メニュー UI の整理（タブ構成・折りたたみ・カメラタブ・音声レベルメータなど）

## 長期記憶の使い方

1. いつもどおり会話する（Gemini API キーが必要）。
2. 会話が途切れてから **約 10 分** 経過すると、記憶が自動で整理・保存される。
3. 名前・趣味などが覚えられ、次回以降の会話で AI が参照する。
4. v0.7 以降は **キャラ** タブの **記憶を見る** で内容を確認できる。
5. 記憶を消したいときは **記憶を削除** を押す（確認後に削除）。

## 必要な環境

| 項目 | 内容 |
| ---- | ---- |
| Aivis Speech Engine | 音声合成に必須 |
| Gemini API キー | `~/.env` の `GEMINI_API_KEY`（v0.7 以降は設定タブからも保存可） |

## アップグレード時の注意

- v0.5.0 からの追加要件はありません（Aivis + Gemini API キーは従来どおり）
- 長期記憶は初回から自動で有効です
- スピーカー利用時は **マイク感度調整** を推奨
- マイクミュートは **Option + スペース**（以前の `M` から変更）
